Privacy Policy

CSP Bypass Search Chrome Extension

Last updated: 16/12/2025

1. Introduction

This Privacy Policy describes how the CSP Bypass Search Chrome Extension ("the Extension", "we", "our") handles information when you use our extension. We are committed to protecting your privacy and being transparent about our data practices.

2. Information We Collect

The Extension is designed with privacy in mind. We collect minimal data and only what is necessary for the extension to function:

2.1 Content Security Policy (CSP) Headers

The Extension reads Content Security Policy headers from web pages you visit. This information is:

  • Stored temporarily in memory (by browser tab ID) while you have the page open
  • Used solely to provide CSP bypass suggestions
  • Never transmitted to external servers
  • Automatically deleted when you close the browser tab

CSP headers are public information that websites send to your browser. We do not collect any personal information from these headers.

2.2 Local Storage

The Extension uses Chrome's local storage to cache data locally on your device:

  • CSP Bypass Data Cache: Public TSV data fetched from GitHub is cached locally for 6 hours to minimize network requests
  • Last Query: Your last search query is stored locally to restore it when you reopen the extension

All data stored locally remains on your device and is never transmitted to external servers (except for fetching the public data file from GitHub, as described below).

3. How We Use Information

The information we collect is used exclusively for the following purposes:

  • To detect and display CSP headers from web pages you visit
  • To search and display relevant CSP bypass suggestions
  • To cache data locally to improve performance and reduce network requests
  • To restore your last search query when you reopen the extension

4. Data Sharing and Third-Party Services

We do not sell, trade, or share your information with third parties. The Extension:

  • Fetches public CSP bypass data from https://raw.githubusercontent.com/renniepak/CSPBypass/refs/heads/main/data.tsv
  • This is a public GitHub repository, and the data fetched is non-personal, technical information about CSP bypass techniques
  • No personal information is transmitted during this fetch
  • Data is cached locally for 6 hours to minimize requests

5. Permissions Explained

The Extension requires the following permissions to function:

  • clipboardWrite: Allows copying CSP bypass payloads to your clipboard when you click on results
  • storage: Used to cache CSP bypass data locally and store your last search query
  • tabs: Required to read CSP headers from the current active tab
  • webRequest: Used to intercept HTTP response headers to detect CSP headers
  • scripting: Used to read CSP headers from HTML meta tags on web pages
  • host_permissions (<all_urls>): Required to read CSP headers from any website you visit

These permissions are necessary for the Extension's core functionality. We do not use these permissions to collect personal information or track your browsing behavior.

6. Data Security

All data collected by the Extension is stored locally on your device using Chrome's secure local storage API. We do not maintain any external servers or databases that store your information.

7. Data Retention

Data is retained as follows:

  • CSP Headers: Stored in memory only while a browser tab is open; deleted immediately when the tab is closed
  • Cached TSV Data: Cached locally for 6 hours, then automatically refreshed from GitHub
  • Last Query: Stored locally until you clear your browser's extension data or uninstall the extension

You can delete all stored data at any time by uninstalling the extension or clearing extension data in Chrome's settings.

8. Your Rights

You have the right to:

  • Uninstall the Extension at any time, which will delete all locally stored data
  • Clear extension data through Chrome's settings (chrome://extensions → Details → Remove)
  • Review this Privacy Policy at any time

9. Children's Privacy

The Extension is intended for security researchers, ethical hackers, and penetration testers. We do not knowingly collect information from children under 13. If you believe we have inadvertently collected such information, please contact us immediately.

10. Changes to This Privacy Policy

We may update this Privacy Policy from time to time. We will notify you of any changes by updating the "Last updated" date at the top of this page. You are advised to review this Privacy Policy periodically for any changes.

12. Purpose and Ethical Use

This Extension is designed for ethical purposes only. It is intended to help:

  • Security researchers identify potential CSP misconfigurations
  • Ethical hackers and penetration testers perform authorized security assessments
  • Web developers understand and improve their Content Security Policy implementations
